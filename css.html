Propriedade Display

display é a maneira que mostra os conteúdo (elementos) da pagina. E existe 4 tipos, são:

Block (Bloco):

a linha todo do conteúdo

altura é defina pelo tamanho do texto

pode usar width e height na propriedade

por padrão não coloca o elemento ao lado

Inline (Em linha):

* por padrão coloca o elemento fica do lado do outro * width e height são definida pelo conteúdo * widht e height não são definidos * comportamento de uma palavra, então remova os enter. para alinha verticalmente use o vertical-align: top // vertical-align: middle; // vertical-align: bottom;

Inline-Block

como no display block fica um conteudo abaixo do outro defenindo width e height, e no inline fica um ao lado do outro sem definição de width e height. No inline-block os conteudo podem ficar um do lado do outro e pode também definir o width e o height

Os elementos que recebem o valor inline-block para a propriedade display obedecem às especicações de dimensão das propriedades height (altura) e width (largura) e ainda permitem que outros elementos sejam exibidos ao seu lado como elementos inline.

Propriedade Position

Assim que você definir o position, logo depois tem que definir as coordenadas de posição, com top, bottom, left e right.

Position: Static

Por padrão os elementos são static, é o tipo um centroavante que fica parado. Se passarmos algum valor para as propriedades de coordenadas, eles não serão respeitados, por não aceitar um posicionamento manual (left, right, top, bottom)

Position: Relative

Por padrão o elemento será renderizado da mesma maneira que o static, aceitando posicionamento manual. O tamanho do seu elemento pai leva em conta o tamanho do elemento relative, porém sem levar em conta seu posicionamento. O pai não sofreria alterações mesmo se o elemento fosse static.

Relative - é relativo ao seu próprio elemento que foi criado quando era static. Assim as coordenadas de posição partirá e moverá da onde foi criado.

    - Abaixo no primeiro quadrado com o position: relative estabelecido, coloquei também um left: 30px.

    - No segundo quadrado com o position: relative estabelecido, coloque também um bottom: 20px e um left: 10px;

    - O Terceiro quadrado não tem position: relative, assim por padrão fica o position: static

Position: Absolute

* Absolute – Toma por base o elemento pai, quem vem antes do elemento. Fica "em baixo" do novo contexto. Assim as coordenadas de posição se move em relação a página. Para respeitar o elemento pai, bastar o pai ser diferente de static * Por denição, o elemento que tem * o modo de posicionamento absolute toma como referência qualquer elemento que seja seu pai na estrutura * doHTMLcujomodo de posicionamento seja diferente de static (que é o padrão), e obedece às coordenadas * de acordo com o tamanho total desse elemento pai. Quando não há nenhum elemento em toda a hierarquia daquele que recebe o posicionamento absolute que seja diferente de static, o elemento vai aplicar as coordenadas tendo como referência a porção visível da página no navegador. Por exemplo: Estilo CSS .quadrado { background: green; height: 200px; width: 200px; } .absoluto { position: absolute; top: 20px; right: 30px; } não tem nenhum elemento como seu “pai” na hierarquia do documento, portanto ele vai alinhar-se ao topo e à direita do limite visível da página no navegador, adicionando respectivamente óþpx e ìþpx nessas direções. Estilos CSS .quadrado { background: green; height: 200px; width: 200px; } .absoluto { position: absolute; top: 20px; right: 30px; } .relativo { position: relative; } Nesse caso, o elemento que recebe o posicionamento absolute é “lho” do elemento que recebe o posicionamento relative na estrutura do documento, portanto, o elemento absolute vai usar como ponto de referência para suas coordenadas o elemento relative e se posicionar óþpx ao topo e ìþpx à direita da posição original desse elemento. Uma conguração de posicionamento vertical (le. ou right) e uma horizontal (top ou bottom) é obrigatória – O elemento será renderizado na posição indicada, porém relativa ao primeiro elemento pai cujo position seja diferente de static ou, se não existir este pai, relativa à página – Seu tamanho não conta para calcular o tamanho do elemento pai

box model e box-sizing O boxmodel é constituído de quatro áreas retangulares: conteúdo (content): aquilo que será exibido espaçamento (padding): distância entre a borda e o conteúdo bordas (borders): quatro linhas que envolvem a caixa (box) margens (margin): distância que separa um box de outro. box-sizing que nos permite trocar o box model que queremos usar. Por padrão, todos os elementos têm o valor box-sizing: content-box o que indica que o tamanho dele é definido pelo seu conteúdo apenas -- em outras palavras, é o tal box model padrão que vimos antes. Mas podemos trocar por box-sizing: border-box que indica que o tamanho agora levará em conta até a borda – ou seja, o width será a soma do conteúdo com a borda e o padding.
