Internet e Afins

Sistema vs Software

Software é um programa que fez bem uma determinada função, exemplo, software de gestão de entrada e saída.

Sistema é um programa que faz bem várias funções, logo, são vários software, exemplo um ERP ou sistema Windows

Tipos de Linguagem

Linguagem de alto nível são linguagens que se aproximam da linguagem humana.

Linguagem de baixo nível são linguagens que se aproximam da linguagem dos computadores.

Algumas linguagens mesmo sendo de alto nível, precisam transformar o código para que assim o computador possa entender, isso se chama compilação, que é o ato de pegar o código fonte que escrever e passar para o computador entender. (00101010001111010101)

Cliente e Servidor

Servidores: são computadores que guardam páginas ou aplicativos. Quando o dispositivo de um cliente quer acessar uma página, uma cópia dela é baixada do servidor à máquina do cliente para ser mostrada no browser.

Clientes: são os usuários típicos da Web conectados à Internet (por exemplo, seu computador conectado ao seu Wi-Fi, ou seu smartphone conectado à sua rede móvel) e programas de acesso à Web disponíveis nesses aparelhos (geralmente um browser como Firefox ou Chrome).

Para entender melhor o papel de cada um, o client-server é a parte do usuário, que acessa um endereço de site no google: www.alinearaujo.dev, quando apertar a tecla enter, o servidor começar a executar diversas tarefas e uma delas é ver em qual IP está o endereço desse site. Assim que o servidor encontra o IP, ele retorna a resposta para que a página web possa abrir

Uma diferença importante para se entender é sobre Front-End e Back-End. Front-end ou client-side, é o que vemos no navegador, e o back-end ou server-side, é o programa rodando no servidor.

Endereço IP e DNS

A internet funciona baseada por protocolo TCP/IP, que é a funcão principal é garantir o padrão de comunicação entre dispositivos diferentes, assim possando trocar mensagens.

Transmission Control Protocol e Internet Protocol são protocolos de comunicação que definem como os dados viajam pela Web. Isso é, como os mecanismos de transporte que te permitem ir ao shopping, fazer um pedido e comprar. Você pode andar e comprar o que você quer, mas nesse caso vamos imaginar que você ligue para a loja, pague seu pedido pelo telefone e espere pela entrega

Essa comunicação pode ser você querendo acessar um site. Então, através de um computador, que tem um único numero, chamado de endereço IP.

Os IPs mais antigos (IPv4) usam 4 octetos, que são conjuntos de 8 bits separados por pontos, totalizando 32 bits por identificador. Ex: 123.45.67.89 = 01111011.00101101.01000011.01011001 Os IPs mais modernos (IPv6), usam 128 bits ao todo (o que é 4x mais bits que o IPv4). Ex: 2001:0db8:85a3:08d3:1319:8a2e:0370:7344

Agora, imagine temos que decorar cada IP para acessar um site especifico? Puts, muito trabalho. Por isso foi criado Domain Name System, o DNS, que é basicamente como "listas telefonicas", criando a ligação entre o IP e o nome do site. Vale lembrar que seu IP muda constatemente.

Quando você digita um endereço web no seu navegador, este procura o real endereço da página web no servidor DNS antes de poder lhe entregar a página. O navegador precisa encontrar em qual servidor web a página está hospedada, para que ele mande mensagens HTTP para o lugar certo.

Protocolo HTTP

Hypertext Transfer Protocol é um protocolo de aplicação que define uma linguagem para clientes e servidores conversarem entre si. É como a linguagem que você usa para fazer seu pedido e falar com a pessoa que está entregando seu pedido.
